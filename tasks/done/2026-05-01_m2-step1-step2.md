## Step 2 — PIE State Machine

### What shipped
- New `Engine/Source/Editor/EditorState.h` — `enum class EEditorState : unsigned char { Editing, Playing, Paused }` + inline `EditorStateToString()`.
- `EditorSubsystem` owns `m_EditorState` + public `GetEditorState()` + transitions:
  - `EnterPlayMode()` — only valid from `Editing`; serializes scene to `EditorTemp/PlaySession.json` via `SceneSerializer`, sets `Playing`.
  - `PauseToggle()` — `Playing ↔ Paused`. No-op + warn in `Editing`. Does **not** re-serialize.
  - `ExitPlayMode()` — only valid from `Playing|Paused`; restores from temp, sets `Editing`.
  - Private `SetEditorState()` is the single transition log point: `OPAAX_CORE_INFO("EditorSubsystem: <from> -> <to>")`.
- `MainMenuBar::DrawPieControls(EditorSubsystem&)` — Play / Pause / Stop right-aligned via `SetCursorPosX(GetWindowWidth() - totalW - margin)`, gated with `BeginDisabled` per state.
- `PlayStopPanel.{h,cpp}` deleted. Member, `m_bShowPlayStop` flag, ref getter, View-menu entry, and `EditorSubsystem::IsPlaying()` getter all removed.
- `ViewportPanel::Draw(EEditorState)` — pushes `ImGuiCol_Border` color (grey/green/amber).
  - **User-applied tweak (2026-05-01)**: explicit `PushStyleVar(ImGuiStyleVar_WindowBorderSize, 3.f)` was commented out in `ViewportPanel.cpp` after the initial implementation; only the color is pushed now. Don't re-add the size push without confirming.
- Play-only gating hook on engine subsystem manager:
  - `IEngineSubsystem::IsPlayOnly() const noexcept` — virtual, default `false`.
  - `EngineSubsystemMgr::UpdateAll(dt, bool bAllowPlayOnly)` and `FixedUpdateAll(dt, bool bAllowPlayOnly)` skip subsystems where `IsPlayOnly() && !bAllowPlayOnly`. Base no-arg overloads kept via `using`-declarations.
  - `CoreEngineApp::Run` computes `bAllowPlayOnly` from `EditorSubsystem::GetEditorState() == Playing` (`#if OPAAX_WITH_EDITOR`); release build hardcodes `true`.
- No production play-only subsystem exists yet — physics (M5) will be the first consumer. To exercise gating before then, override `IsPlayOnly()` on a throwaway subsystem.
